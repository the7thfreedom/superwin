# Contributing to SuperWin

Thank you for your interest in contributing!

## Guidelines

1. **Stay upstream-compatible** — Keep the codebase compatible with [superset-sh/superset](https://github.com/superset-sh/superset) where possible.
2. **Isolate Windows changes** — Windows-specific code should be clearly marked and documented.
3. **License compliance** — All contributions must be compatible with the **Elastic License 2.0 (ELv2)**.
4. **Document changes** — Update `docs/windows-changes.md` for any platform-specific modifications.

## Getting Started

1. Fork the repo and create a feature branch
2. Make your changes with clear commit messages
3. Open a Pull Request describing what you changed and why

## Reporting Issues

Open an [Issue](../../issues) for bugs, questions, or suggestions.
